# The Weekly Thing Publishing Process

Studio exists to help Jamie publish each issue of **The Weekly Thing** well.
The first-class object is a newsletter issue. The active assistant is Eddy. The
primary work surface is the tailnet Studio web app.

Everything else is secondary until this loop works reliably.

## Scope

In scope:

- assemble the current issue
- sync source material into issue rows
- edit authored atoms in Studio
- run Eddy review and follow-up helpers
- render, package, publish, and put the issue to bed

Out of scope for the active product:

- additional agents
- blog-post production
- podcast production as a separate workflow
- generic projects
- seeds, gardening, campaigns, or proactive slate checks

Audio for the newsletter issue can remain in the issue publish path. That is
not the same as reviving podcast production as a separate Studio product.

## State Model

The active issue window carries `phase = build` or `phase = publish`.

| Transition | Trigger | Effect |
|---|---|---|
| none to build | Start issue in Studio | Opens the active issue window and makes the issue editable |
| build to publish | Mark built in Studio | Moves the issue to packaging and runs Eddy's publish-prep jobs |
| publish to build | Reopen in Studio | Returns the issue to editing |
| publish to closed | Put to bed in Studio | Files the issue, closes the active window, and prepares Studio for the next issue |

Content is DB-resident. The preview renders from current DB state. Email,
website, transcript, and audio artifacts are generated by the publish action
that needs them.

## Operator Surface

The Studio web app is the product surface:

- issue overview
- source sync
- atom editor
- live preview
- Eddy review comments
- package/publish controls
- put-to-bed action

Discord remains useful for Eddy notes, follow-ups, and repair commands, but it
is not the primary production UI.

## Boundaries

`apps/workshop_bot` owns the newsletter studio runtime. The Librarian API and
retrieval contract remain part of this mono repo but are a separate subsystem.
Do not change Librarian contracts casually while doing Studio assistant work.
