# Program — Campaigns

*Grow and engage the audience.* A **standing program**, not a phase — ongoing, spanning many
issues. Overview: [`../publishing-process.md`](../publishing-process.md).

**Owner:** Marky · **Channel:** `#promotion`

## Two objectives

- **Engagement** — get existing readers to share / discuss (the r/WeeklyThing megathread, LinkedIn,
  per-link threads).
- **Acquisition** — bring in new readers via **paid placements** (e.g. Dense Discovery ad slots).

## Tools

- **Syndication copy** drafted by `promotion-prep` (never auto-posted).
- **`?ref=` tracking** — Tinylytics referrer traffic + Buttondown ref signups, polled by
  `daily-metrics`.
- The **campaigns ledger** — `/marky campaign {add,edit,report,copy,sunset}` (a campaign records its
  `ref`, expected vs actual signups/traffic, and the creative copy that ran).

## Touchpoints

- **Share** (the per-issue phase) — syndicating each published issue. See
  [`../phases/share.md`](../phases/share.md).
- **Standalone placements** — ad buys that aren't tied to any one issue and run on their own
  cadence. These live entirely in the program (the ledger + ref tracking), not in any phase card.

> **Marky spans both layers:** he owns the per-issue **Share phase** *and* runs this standing
> program. The phase is "promote *this* issue"; the program is "grow the audience over time."
