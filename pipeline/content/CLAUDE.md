# pipeline/content/ тАФ project memory

The unified content CLI lives here. There's no README тАФ this directory is operator-only tooling.

## The CLI: `pipeline/content/content.py`

One entrypoint, subcommands. Wrapped by Makefile targets where useful.

| Subcommand | Make target | What it does |
|---|---|---|
| `build` | `make build` | Read `data/issues/{N}/{archive.md, metadata.json, links.json}` and write `apps/site/archive/{N}.md` + `apps/site/_data/emails.json`. Idempotent. Generated files carry an inline `<!-- Generated by тАж -->` notice. |
| `stats` | `make stats` | Refresh `apps/site/_data/stats.json` (subscriber count + premium count from Buttondown, Stripe balance, year-to-date donations). Requires `BUTTONDOWN_API_KEY` + `STRIPE_API_KEY`. |
| `publish --issue N` | тАФ | POST/PATCH `buttondown.md` to Buttondown as a draft. Idempotent via `buttondown_id` stored in `metadata.json`. Workshop_bot's `send-to-buttondown` wraps this; CLI form is the manual fallback. Use `--dry-run` first on a low-stakes issue. |

The build is the canonical site flow: `data/issues/` is editorial truth, `apps/site/archive/` is generated. Hand-edits to the generated files get clobbered on the next build.

## Other modules

- **`domain_exclusions.py`** тАФ the canonical list of domains stripped from the `domains[]` front-matter field + the home-page domain stats (own domains, Buttondown, image CDNs, URL shorteners, social, Wikipedia, YouTube, generic CDNs). `apps/workshop_bot/tools/avoid_domains.py` is a hand-maintained *copy* used for Pinboard pre-filtering тАФ keep loosely in sync; an exact match isn't required since the bot's copy gates Linky's scan, nothing it touches ships.
- **`refresh_marketing_copy.py`** тАФ generates `apps/site/_data/voiceSamples.json` (home-page pull-quotes) + rewrites its own persistent context, **`marketing-brief.md`** (in this dir; the canonical *human* voice doc is `docs/voice-and-style.md`, which the brief should track). Stratified sample of ~32 issues over the last 2 years; Sonnet picks 3-5 verbatim pull-quotes. Machine-verified verbatim against issue bodies before writing. **Not in CI** тАФ explicit `make refresh-copy-dry` / `make refresh-copy`. Expected cost: $0.20-0.40 per run.
- **`fetch_emails.py`** тАФ pulls subscriber events from Buttondown. Operator script; not in the build path.
- **`generate_descriptions.py`** тАФ backfill helper that generates meta descriptions for archive issues missing one. One-off; mostly retired.

## Build flow (what `make build` does end-to-end)

1. `pipeline/content/content.py build` writes `apps/site/archive/{N}.md` + `_data/emails.json`.
2. Audio fields (`audio_url`, `audio_duration`, `audio_byte_size`, `audio_voice`) get injected from `data/audio/manifest.json` per issue.
3. 11ty renders the site (`npx @11ty/eleventy --config apps/site/eleventy.config.js`).
4. Pagefind indexes the built HTML (`npx pagefind --site _site`).

GitHub Actions runs the same sequence on push to main; `continue-on-error` on stats + audio + corpus upload so a downstream hiccup doesn't block the deploy.

## Conventions

- **`content.py build` is idempotent.** Re-running with no source changes produces no diff. Don't add side effects that violate this.
- **The `<!-- Generated by тАж -->` notice** belongs on every generated archive file. It's how reviewers know not to hand-edit.
- **Print progress to line-buffered stdout** (`sys.stdout.reconfigure(line_buffering=True)`) тАФ the default block-buffering hides progress when piped. (Memory: `feedback_unbuffered_output.md`.)
- **Domain exclusion** тАФ when adding a domain, update both `pipeline/content/domain_exclusions.py` AND `apps/workshop_bot/tools/avoid_domains.py`. The two should be loosely consistent.
- **Stats refresh hits Buttondown's `/api/subscribers` + Stripe's `/v1/balance` + `/v1/balance_transactions`.** Both have generous rate limits but Stripe pagination can be slow for full year-to-date тАФ cached at the script level when possible.
