I ran across [BlogFlock](https://blogflock.com), a new service by [Raphael Kabo](https://raphael.computer/) that takes a social approach to RSS feeds, and decided to give it a look. BlogFlock is entirely built on open RSS feeds and allows users to create lists of RSS feeds that others can then follow. Think of it as a way to build a custom timeline using RSS in the open. It isn't a Feed Reader because it isn't just _your_ feed or list and there is no read or unread status. It is more in the spirit of a "river of news" feed that anyone can use.

I was having fun with it and decided to create a list I've always wanted for [Manuel Moreale](https://manuelmoreale.com)'s excellent **[People & Blogs](https://peopleandblogs.com)** project. I created a [BlogFlock list for People & Blogs](https://blogflock.com/list/Qj5Gq) and yes, I actually added all 66 blogs to the list (some exceptions noted below). That wasn't super fun but it is done and adding each new one is simple enough.

I had previously thought of doing something like this in Feedbin but it isn't public so it kind of missed the point. I wanted something more like a "river of news" for it. So BlogFlock was a perfect approach.

It seems to be working really well and I like the way it works. Check out the **[People & Blogs List](https://blogflock.com/list/Qj5Gq)** and give it a look.

Some notes for BlogFlock.

1. You should be able to subscribe to the RSS feed for the list itself. In this way BlogFlock could be an aggregator and then I could add that lists aggregated RSS feed to my feed reader.
2. You for sure should be able to create a list by importing an OPML file.
3. Lists should be exportable via OPML as well.
4. The description for a blog in a list is hopefully scoped just to that list. So if I add the same feed to two different lists I would want a different description for each one.
5. You should be able to manually reorder the blogs in a list.
6. It could handle posts without titles a bit better.
